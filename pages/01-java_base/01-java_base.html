1.1 重载和重写的区别

重载 ● 重载是指在同一个类中存在多个同名方法，但这些方法的参数列表不同（参数类型、个数或者顺序不同）。重载使得你可以根据不同的参数使用相同的方法名，增加程序的可读性以及方法的灵活性。

public class CalculatorUtil {
    // 加法方法用于整数
    public int add(int a, int b) {
        return a + b;
    }

    // 加法方法重载用于浮点数
    public double add(double a, double b) {
        return a + b;
    }
}

重写 ● 重写是指子类中定义一个方法，该方法的签名（方法名和参数列表）与父类中的某个方法的签名完全相同，但有自己的实现。允许子类根据需要实现或修改从父类继承的行为。

public class Animals {
    public void sound() {
        System.out.println("动物叫");
    }
}

public class Dog extends Animals {
    // 重写父类Animal类的sound方法
    @Override
    public void sound() {
        System.out.println("犬吠");
    }
}

重载发生在同一个类中，利用同一个方法名处理不同类型或数量的参数，是一种水平关系。 重写则是在继承关系中发生，子类重新定义从父类继承来的方法以适应特定的需求，是一种垂直关系。

1.2 ==和equals()的区别

== 对于基本数据来说比较的是值； 对于引用数据类型比较的是对象的内存地址；

equals() equals()只能用于比较两个对象是否相等。equals 方法在 Object 类中默认行为是使用 == 比较对象的引用，但通常在各种类中被重写以实现内容比较。

public boolean equals(Object obj) {
    return (this == obj);// 比较内存地址是否相等
}

类没有重写equals()时，比较两个对象，相当于比较两对象的内存地址值是否相等，类似于“==”。 类重写了equals()时，比较的是两对象的值(属性)是否相等。比如我们熟悉的String类就重写了equals()方法。 String类中重写equals()

public boolean equals(Object anObject) {
    // 首先检查是否为同一对象的引用，如果是则直接返回true
    if (this == anObject) {
        return true;
    }
    // 检查传入的对象是否为String类型的实例
    if (anObject instanceof String) {
        // 将传入的对象强制转换为String类型
        String anotherString = (String)anObject;
        // 获取当前字符串的长度
        int n = value.length;
        // 比较当前字符串和另一个字符串的长度，如果不同，则直接返回false
        if (n == anotherString.value.length) {
            // 获取当前字符串内部的字符数组
            char v1[] = value;
            // 获取另一个字符串内部的字符数组
            char v2[] = anotherString.value;
            // 从第一个字符开始比较
            int i = 0;
            // 遍历所有字符进行比较
            while (n-- != 0) {
                // 如果在任何位置发现字符不同，则返回false
                if (v1[i] != v2[i])
                    return false;
                // 移动到下一个字符
                i++;
            }
            // 如果所有字符都相同，则返回true
            return true;
        }
    }
    // 如果传入的对象不是String类型或长度不匹配，则返回false
    return false;
}
